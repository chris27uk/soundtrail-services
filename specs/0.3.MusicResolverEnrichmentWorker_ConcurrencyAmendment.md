# Enrichment Worker Specification Amendment
## Distributed Provider Rate Limiting & Concurrency

> Historical note: [1.0.SoundtrailServicesCanonicalSpecification.md](/Users/chris/Developer/Soundtrail.Services/specs/1.0.SoundtrailServicesCanonicalSpecification.md) is the current authoritative spec. This amendment is future design input unless adopted into the canonical specification.

This amendment updates the `MusicResolverEnrichmentWorkerSpecification.md`.

---

# Problem

The existing specification defines:

- maxPerMinute
- maxPerHour
- maxPerDay
- maxConcurrent

However, it does not define how concurrency is enforced across multiple worker instances.

A local `SemaphoreSlim` is not sufficient because multiple Azure Container App instances would each have their own semaphore.

Provider limits must be global.

---

# Design Principle

Provider limits are distributed resources.

Workers must acquire:

1. provider concurrency slot
2. provider rate-limit budget

before making any third-party provider call.

---

# Provider Budget Model

Each provider has four independent controls:

```text
Minute Tokens
Hour Tokens
Day Tokens
Concurrent Execution Slots
```

Example:

```json
{
  "AppleMusic": {
    "MaxPerMinute": 5,
    "MaxPerHour": 100,
    "MaxPerDay": 500,
    "MaxConcurrent": 1
  }
}
```

# New Ports

```csharp
public interface ProviderConcurrencyPort

    Task<ConcurrencyLease?> TryAcquireAsync(
        Provider provider,
        CancellationToken ct);

    Task ReleaseAsync(
        ConcurrencyLease lease,
        CancellationToken ct);
}
```

## ProviderBudgetPort

```csharp
public interface ProviderBudgetPort
{
    Task<ProviderBudgetDecision> TryConsumeAsync(
        Provider provider,
        ProviderRateLimitPolicy policy,
        CancellationToken ct);
}
```

# Worker Flow

```text
Receive Job
  ↓
Check Circuit Breaker
  ↓
Acquire Concurrency Lease
  ↓
Consume Provider Budget
  ↓
Call Provider
  ↓
Release Lease
```

If either step fails:

```text
Acquire Lease Failed
  ↓
Retry Later

Budget Exhausted
  ↓
Retry Later
```

### Example

```text
PartitionKey = provider
RowKey       = AppleMusic

LeaseOwner = worker-7
LeaseId = abc123
ExpiresAt = 2026-01-01T12:00:00Z
```

Workers must use optimistic concurrency (ETags) when acquiring leases.

Leases must expire automatically if a worker crashes.

# Scheduler Changes

The scheduler must check:

```text
Provider Enabled
Provider Circuit Closed
Provider Budget Available
Concurrency Capacity Likely Available
```

before enqueueing expensive jobs.

This is only an optimisation.

The worker remains the final authority.

---

# Worker Changes

Workers must:

```text
Acquire Lease
Consume Budget
Call Provider
Release Lease
```

Provider calls must never occur before both checks succeed.

# Additional Tests

Required tests:

1. Multiple workers cannot exceed Apple concurrency limit.
2. Lease expires after worker crash.
3. Lease can be reacquired after expiry.
4. Budget limit is enforced globally.
5. Budget consumption is atomic.
6. Worker never calls provider without lease.
7. Worker never calls provider when budget exhausted.
8. Scheduler skips jobs when provider disabled.
9. Scheduler skips jobs when circuit open.
10. Scheduler prefers providers with available budget.

---

# Acceptance Criteria Additions

The enrichment worker implementation is complete only when:

- provider concurrency is enforced globally
- provider rate limits are enforced globally
- concurrency works across multiple container instances
- provider calls require a lease and budget
- leases expire automatically
- worker crashes do not permanently consume concurrency slots
- tests prove concurrency enforcement
